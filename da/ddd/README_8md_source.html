6 Description
7 -------------
8 
9 **PURIFY** is an open-source collection of routines written in `C++` available under the [license](#license) below. It implements different tools and high-level to perform radio interferometric imaging, _i.e._ to recover images from the Fourier measurements taken by radio interferometric telescopes.
10 
11 **PURIFY** leverages recent developments in the field of compressive sensing and convex optimization. Low-level functionality to solve the resulting convex optimisation is factored into the open-source companion code, [**SOPT**](https://github.com/astro-informatics/sopt), also written by the authors of **PURIFY**. For further background please see the [reference](#references-and-citation) section.
12 
13 This documentation outlines the necessary and optional [dependencies](#dependencies-installation) upon which **PURIFY** should be built, before describing [installation](#installing-and-building-PURIFY), [testing](#testing) and [usage](#usage) details. [Contributors](#contributors), [references](#references-and-citation) and [license](#license) information then follows.

15 Dependencies installation
16 -------------------------
17 
18 **PURIFY** is written in `C++11`. Pre-requisites and dependencies are listed in following and minimal versions required are tested against `Travis CI` meaning that they come natively with OSX and the Ubuntu Trusty release. These are also the default ones fetched by `CMake` (an internet connection is required for this).
19 
20 `C++` minimal dependencies:
21 
22 - [CMake](http://www.cmake.org/) v3.9.2 A free software that allows cross-platform compilation
23 - [GCC](https://gcc.gnu.org) v7.3.0 GNU compiler for `C++`
24 - [UCL/GreatCMakeCookOff](https://github.com/UCL/GreatCMakeCookOff) Collection of `CMake` recipes.
25  Downloaded automatically if absent
26 - [fftw3](www.fftw.org): Fastest Fourier Transform in the West
27 - [Eigen3](http://eigen.tuxfamily.org/index.php?title=Main_Page) v3.2.0 (Trusty) Modern `C++` linear algebra.
28  Downloaded automatically if absent.
29 - [tiff](http://www.libtiff.org/) v4.0.3 (Trusty) Tag Image File Format library
30 - [astro-informatics/sopt](https://github.com/astro-informatics/sopt) v* Sparse Optimization
31  Compressed Sensing library. Downloaded automatically if absent.
32 - [cfitsio](http://heasarc.gsfc.nasa.gov/fitsio/fitsio.html) v* Library of `C` and `Fortran` subroutines
33  for reading and writing data files in FITS (Flexible Image Transport System) data format.
34  Downloaded automatically if absent.
35 - [casacore](http://casacore.github.io/casacore/) - Optional - Needed to interface with measurement
36  sets. The main purify program requires this library (and its dependencies)

38 - [MPI](https://www.open-mpi.org) v3.1.1 (Trusty) - Optional - Parallelisation paradigm to speed up operations.
39 - [spdlog](https://github.com/gabime/spdlog) v* - Optional - Logging library. Downloaded automatically if
40  absent.
41 - [Catch2](https://github.com/catchorg/Catch2) v2.3.0 - Optional - A `C++`
42  unit-testing framework only needed for testing. Downloaded automatically if absent.
43 - [google/benchmark](https://github.com/google/benchmark) `v1.3.0` - Optional - A `C++`
44  micro-benchmarking framework only needed for benchmarks. Downloaded automatically if absent.

46 Installing and building PURIFY
47 -------------------------------------
48 
49 **PURIFY** can be installed through the software packet manager on Linux Debian distributions:
50 
51 ```
52 apt-get install purify
53 ```
54 
55 Alternatively, you can build **PURIFY** entirely from the source code. Once the mandatory dependencies are present, `git clone` from the [GitHub repository](https://github.com/astro-informatics/purify):
56 
57 ```
58 git clone https://github.com/astro-informatics/purify.git
59 ```
60 
61 Then, the program can be built with standard `CMake` command:
62 
63 ```
64 cd /path/to/code
65 mkdir build
66 cd build
67 cmake -DCMAKE_BUILD_TYPE=Release ..
68 make
69 ```
70 
71 To test everything went all right:
72 
73 ```
74 cd /path/to/code/build
75 ctest .
76 ```
77 
78 To install in directory `/X`, with libraries going to `X/' do:
79 
80 ```
81 cd /path/to/code/build
82 cmake -DCMAKE_INSTALL_PREFIX=/X ..
83 make install
84 ```

96 Usage
97 ------
98 
99 The main purify executable lives either in the build directory or in the in the `bin` subdirectory
100 of the installation directory. `purify` has one required argument, it a string for the file path of the config file containg the settings.
101 
102 `purify path/to/config.yaml`.
103 
104 A template with a description of the settings can be found at https://github.com/astro-informatics/purify/blob/master/data/config/config.yaml.
105 This can also be found in the downloaded repo at `data/config/config.yaml`. When `purify` runs a directory will be created, and the output images will be saved and time-stamped. Additionally, a config file with the settings used will be saved and time-stamped, helping for reproducibility and book-keeping.

129 CMake Tips
130 ----------
131 
132 It is possible to tell ``CMake`` exactly which libraries to compile and link against. The general
133 idea is to add ``-DVARIABLE=something`` to the command-line arguments of CMake. CMake can be called
134 any number of times: previous settings will not be overwritten unless specifically
135 requested. Some of the more common options are the following:
136 
137 - ``CMAKE_PREFIX_PATH``: CMake will look in "CMAKE_PREFIX_PATH/lib"
138  for libraries, "CMAKE_PREFIX_PATH/include" for headers, etc.
139 - ``FFTW3_LIBRARIES``, ``FFTW3_INCLUDE_DIR``
140 - ``BLAS_INCLUDE_DIRS``, ``BLAS_LIBRARIES``
141 
142 All these variables and more can be found and modified in the ``CMakeCache.txt`` file in the build
143 directory. There are extra CMake options sepcific to purify. ``-Ddompi=ON`` will turn MPI on in the build, ``-Dopenmp=ON`` will turn openmp on for the build. ``-Dtests=ON`` will make sure tests are built.
144 
145 ``ctest`` should be run to make sure the unit tests pass.
